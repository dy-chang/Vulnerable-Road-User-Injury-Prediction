    "## Step 1 · Data Cleaning & Target Variable Design\n",
    "\n",
    "### Why the original target (`has_injury`) was weak\n",
    "\n",
    "A first attempt might use:\n",
    "\n",
    "```python\n",
    "has_injury = 1  # if anyone was injured  (positive rate ≈ 40%)\n",
    "```\n",
    "\n",
    "**Problems:**\n",
    "\n",
    "| Issue | Explanation |\n",
    "|---|---|\n",
    "| **Too easy** | 40% positive rate — model can score ~60% accuracy by predicting the majority class |\n",
    "| **Not specific** | \"Someone was injured\" includes minor fender-benders with soft-tissue claims |\n",
    "| **No policy hook** | Doesn't distinguish *who* was hurt — a key question for Vision Zero |\n",
    "| **Low AUC ceiling** | With only temporal + borough features, AUC tops out at ≈ 0.57–0.62 |\n",
    "\n",
    "### The fix: design a target with domain meaning\n",
    "\n",
    "> **`vru_injury`** = 1 if a pedestrian **or** cyclist was injured in the crash\n",
    "\n",
    "| Property | `has_injury` | `vru_injury` |\n",
    "|---|---|---|\n",
    "| Positive rate | 40% | **14.9%** (imbalanced, realistic) |\n",
    "| Policy relevance | Generic | Vision Zero · Complete Streets |\n",
    "| Feature signal | Weak | Stronger — vehicle type, multi-vehicle now predictive |\n",
    "| Achievable AUC | 0.57–0.62 | **0.82** |\n",
    "\n",
    "> **Takeaway:** In applied transportation ML, *problem framing* matters more than algorithm choice.  \n",
    "> A well-defined target tied to a real decision is always better than a convenient proxy."

    "---\n",
    "## Step 3 · Feature Engineering\n",
    "### Adding transportation-domain knowledge\n",
    "\n",
    "Rather than trying more complex algorithms, we add **domain-informed features** that encode\n",
    "real-world mechanisms of VRU injury:\n",
    "\n",
    "| Feature | Rationale |\n",
    "|---|---|\n",
    "| `n_vehicles`, `multi_veh` | Multi-vehicle collisions create more chaotic intersection conditions — higher VRU exposure |\n",
    "| `vt_bike` | Crashes *involving* a bike or moped are almost always VRU events by definition |\n",
    "| `vt_taxi`, `vt_bus` | High spatial correlation with pedestrian-dense zones (midtown, transit hubs) |\n",
    "| `vt_truck` | Blind-spot crashes — known risk factor for cyclists and pedestrians |\n",
    "| `hour_sin / hour_cos` | **Cyclic encoding** prevents the model from treating 11 PM (H23) and midnight (H0) as distant |\n",
    "| `dow_sin / dow_cos` | Same cyclic logic for day-of-week |\n",
    "| `cf_alcohol` | Alcohol/drug-impaired driving — known severity multiplier for pedestrian outcomes |\n",
    "| `cf_speed` | Unsafe speed — primary determinant of pedestrian injury severity |\n",
    "| `cf_distraction` | Distracted driving — strongly associated with failure to yield to pedestrians |\n",
    "\n",
    "> **This is domain feature engineering:** each feature encodes a transportation safety mechanism,\n",
    "> not just a data column. This is what separates a transportation data scientist from a generic ML practitioner."
   ]
  },

    "# ── Vehicle type flags ──────────────────────────────────────────────────────────\n",
    "vtc1 = df[\"vehicle_type_code1\"].fillna(\"Unknown\").str.lower()\n",
    "df[\"vt_bike\"]  = vtc1.str.contains(\"bike|moped|scooter|motor\", na=False).astype(int)\n",
    "df[\"vt_taxi\"]  = vtc1.str.contains(\"taxi|livery\", na=False).astype(int)\n",
    "df[\"vt_bus\"]   = vtc1.str.contains(\"bus|van\", na=False).astype(int)\n",
    "df[\"vt_truck\"] = vtc1.str.contains(\"truck|pick-up|tractor\", na=False).astype(int)\n",
    "df[\"vt_suv\"]   = vtc1.str.contains(\"station wagon|sport utility|suv\", na=False).astype(int)\n",
    "\n",
    "# ── Multi-vehicle involvement ──────────────────────────────────────────────────\n",
    "df[\"n_vehicles\"] = df[[\"vehicle_type_code1\", \"vehicle_type_code2\",\n",
    "                        \"vehicle_type_code_3\"]].notna().sum(axis=1)\n",
    "df[\"multi_veh\"]  = (df[\"n_vehicles\"] >= 2).astype(int)\n",
    "\n",
    "# ── Cyclic time encoding ───────────────────────────────────────────────────────\n",
    "df[\"hour_sin\"] = np.sin(2 * np.pi * df[\"hour\"] / 24)\n",
    "df[\"hour_cos\"] = np.cos(2 * np.pi * df[\"hour\"] / 24)\n",
    "df[\"dow_sin\"]  = np.sin(2 * np.pi * df[\"dow\"]  / 7)\n",
    "df[\"dow_cos\"]  = np.cos(2 * np.pi * df[\"dow\"]  / 7)\n",
    "\n",
    "# ── Contributing factor flags (simple NLP on free-text field) ────────────────\n",

    "df[\"cf_fatigue\"]     = cf.str.contains(\"fatigue|sleep|ill\", na=False).astype(int)\n",
    "df[\"cf_speed\"]       = cf.str.contains(\"speed|unsafe|traffic control\", na=False).astype(int)\n",
    "\n",
    "RICH_FEATS = (\n",

    "    [\"vt_taxi\", \"vt_bus\", \"vt_truck\", \"vt_bike\", \"vt_suv\"] +\n",
    "    [\"multi_veh\", \"n_vehicles\"] +\n",
    "    [\"cf_distraction\", \"cf_alcohol\", \"cf_fatigue\", \"cf_speed\"]\n",
    ")\n",
    "\n",
